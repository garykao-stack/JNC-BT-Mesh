# JNC BT-Mesh LED 燈號說明（維護人員用）

對象：現場安裝 / 維護人員
適用機型：JNC-BT-Mesh（板 **JNC V1.0**，三色 LED，高電位點亮）
韌體：v1.41 以上適用（燈號邏輯長期未變）

---

## 1. 三顆 LED 對應

| 板上順序 | 韌體代號 | 顏色 | 接腳 |
|----------|----------|------|------|
| LED1 | LED0 / LED_RED | 🔴 紅 | PD14 |
| LED2 | LED1 / LED_BLUE | 🔵 藍 | PD15 |
| LED3 | LED2 / LED_GREEN | 🟢 綠 | PB11 |

> 韌體內部編號為 LED0~LED2（紅/藍/綠）。若板上絲印標 LED1/LED2/LED3，依序對到 紅→藍→綠。

---

## 2. 三顆燈各自主要意義（速記）

- 🔴 **紅** — 角色＝**感測端 (Server)**。工作中恆亮、睡眠熄滅；中繼端收到資料時會閃。
- 🔵 **藍** — 角色＝**中繼/接收端 (Client)** 恆亮；組網 / IVI 更新 / 維修模式時閃；**回報失敗時持續閃（最常見的故障指示）**。
- 🟢 **綠** — **RS485 / 感測資料活動**：收 sensor 資料時亮，掃描偵測或來源切換時閃。

---

## 3. 開機與組網（依時間順序）

| 階段 | 🔴 紅 | 🔵 藍 | 🟢 綠 | 代表狀態 |
|------|-------|-------|-------|----------|
| 開機自檢 | 亮 1 秒 | 亮 1 秒 | 亮 1 秒 | 三顆同時亮 1 秒後全滅＝電源/LED 正常 |
| 未組網 | 閃 | 閃 | 滅 | 紅+藍一起閃＝**尚未加入網路**，等待 App 佈建 |
| 開始組網 | 滅 | 恆亮 | 滅 | 藍燈恆亮＝App 已開始佈建 |
| 組網中 | 交替閃 | 交替閃 | 滅 | 紅↔藍**交替**閃爍＝佈建進行中 |
| 組網完成 | 進入下方「運轉中」 | | | 完成後裝置會**自動重開一次**套用角色，屬正常 |
| 設定模式 | 滅 | 恆亮 | 滅 | Win-Utility 參數設定模式 |

---

## 4. 運轉中燈號 → 原因 → 處置

| 燈號表現 | 可能原因 | 建議處置 |
|----------|----------|----------|
| 🔴 紅 恆亮 | 感測端**工作/喚醒中**（正常） | 無須處理 |
| 🔴 紅 熄滅（且非未組網） | 感測端**睡眠中**（省電，正常） | 無須處理 |
| 🔵 藍 恆亮 | 此節點為**中繼/接收端 (Client)**（正常） | 無須處理 |
| 🟢 綠 亮/閃 | 正在**接收 RS485 / 感測器資料**（正常） | 無須處理 |
| 🟢 綠 連閃 3 次 | 感測資料來源切換（內建 ↔ RS485） | 無須處理，確認接的 sensor 正確即可 |
| 🔵 藍 **持續每 0.5 秒閃** ⚠️ | **Mesh 回報失敗**（資料送不出去 / 收不到回應） | 見下方「§5 回報失敗排查」 |
| 🔴+🔵 一起閃（已裝很久仍如此） | 一直停在**未組網** | 用 App 重新佈建；確認裝置在網路範圍內 |
| 🔵 藍 閃（短暫） | IV Index 更新中 或 維修模式啟用中 | 屬暫時狀態，數分鐘內恢復；維修模式最長 60 分自動結束 |
| 三顆全暗、且無任何反應 | 無電 / 未開機 / 韌體當機 | 檢查供電；必要時重新上電。裝置有看門狗（16 秒）會自動重啟 |

---

## 5. 回報失敗排查（🔵 藍燈持續閃）

藍燈每 0.5 秒閃＝節點把資料送上 Mesh 失敗，依序檢查：

1. **訊號範圍 / 中繼**：與上層節點或中繼點距離是否過遠、中間是否有金屬遮蔽。
2. **網路是否健在**：中繼端 (Client) 是否正常運轉、Mesh 網路是否還在。
3. **重新上電**：對該節點重新上電，觀察是否恢復（藍燈停止閃即恢復正常）。
4. 仍持續閃 → 記錄站號回報原廠。

| 看燈 | 角色 |

| 🔴 紅 為主（亮/睡眠切換、收資料閃） | **感測端 (Server)** |
| 🔵 藍 恆亮 | **中繼/接收端 (Client)** |
